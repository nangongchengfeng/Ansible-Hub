# Ansible Job Platform v1 PRD

Status: needs-triage

## Problem Statement

当前企业使用 Ansible 进行自动化运维时面临以下问题：
- 静态 inventory 文件管理繁琐，难以按业务组织主机
- 私钥和凭证分散在各运维人员机器上，存在安全隐患
- 没有统一的剧本和脚本版本管理，变更无法追溯
- 缺少高危命令拦截机制，容易发生误操作
- 没有作业调度和审计能力，无法满足合规要求
- 难以将自动化能力赋能给开发和测试人员

## Solution

构建一个企业级 Ansible 自动化作业平台，提供：
- 树状业务节点组织的主机资产管理
- 集中加密存储的 SSH 凭证管理
- 版本化的剧本和脚本管理
- 主动式高危命令拦截
- 灵活的作业模板和调度能力
- 完整的审计日志和运营分析
- 基于角色的细粒度权限控制

## Implementation Decisions

### 技术栈

- **后端**：FastAPI + SQLAlchemy + Alembic + Celery + Redis + ansible-runner + Paramiko + Fernet
- **前端**：Vue 3 + Element Plus + Vue Router + Pinia
- **数据库**：MySQL 8
- **认证**：JWT
- **代码规范**：Black + isort + flake8（后端），ESLint + Prettier（前端）
- **Git 工作流**：GitHub Flow

### 核心模块

1. **认证与授权模块**
   - 用户管理、角色管理、JWT 认证
   - 业务节点权限检查
   - 作业模板权限检查

2. **资产与连接模块**
   - 业务节点树管理（含权限）
   - 主机管理
   - 系统用户管理（含加密存储）
   - 网关管理
   - 连接配置解析（含继承逻辑）

3. **剧本与脚本模块**
   - 剧本 CRUD + 版本管理
   - 脚本 CRUD + 版本管理
   - 版本 diff 对比
   - 回滚功能

4. **安全模块**
   - 命令过滤规则管理
   - 命令匹配检查
   - 审计日志记录

5. **作业执行模块**
   - 临时作业提交
   - WebSocket 实时输出
   - 作业取消
   - Ansible 调用封装
   - 网关代理连接

6. **作业模板与调度模块**
   - 作业模板 CRUD
   - Cron 调度配置
   - Celery Beat 定时触发
   - 并发控制

7. **历史与分析模块**
   - 作业历史查询
   - 日志存储与检索
   - 运营分析统计
   - 重做功能

### 关键接口

- **连接服务接口**：给定主机 ID，返回完整连接配置（含继承的网关、SSH 端口等）
- **命令过滤接口**：给定命令，返回是否被阻止及匹配的规则
- **作业提交接口**：提交作业，返回作业执行 ID
- **版本管理接口**：获取历史版本列表、版本详情、版本 diff

### 数据模型

- User（用户）
- Role（角色）
- BusinessNode（业务节点，含自引用外键）
- BusinessNodePermission（业务节点权限）
- Host（主机）
- SystemUser（系统用户，敏感字段加密）
- Gateway（网关）
- Script（脚本）
- ScriptVersion（脚本版本）
- Playbook（剧本）
- PlaybookVersion（剧本版本）
- CommandFilterRule（命令过滤规则）
- JobTemplate（作业模板）
- JobTemplatePermission（作业模板权限）
- JobExecution（作业执行）
- Task（任务）
- AuditLog（审计日志）

## Testing Decisions

### 测试策略

- 只测试外部行为，不测试实现细节
- 重点测试核心业务逻辑：权限检查、连接配置继承、命令过滤匹配、版本管理
- 使用测试数据库进行集成测试

### 测试范围

- **认证与授权模块**：全面测试，这是安全基础
- **资产与连接模块**：重点测试连接配置继承逻辑
- **剧本与脚本模块**：测试版本管理和回滚功能
- **安全模块**：全面测试命令过滤匹配逻辑
- **作业执行模块**：测试提交和取消逻辑，实际 Ansible 执行可 Mock
- **作业模板与调度模块**：测试调度配置逻辑
- **历史与分析模块**：测试基本查询功能

## Out of Scope

以下功能不在 v1 范围内：

- 主机连接检测（主动探测）
- 主机批量导入（Excel/CSV）
- 作业执行结果通知（邮件/钉钉/企业微信）
- 资源回收站（软删除）
- 标签系统
- 自定义 SSH 配置
- 多环境支持
- 高可用部署
- 插件系统

## Further Notes

- 所有敏感数据（私钥、密码）使用 Fernet 加密存储，密钥存储在环境变量中
- 审计日志不可删除
- 剧本/脚本删除时级联删除所有历史版本
- 回滚操作创建新版本，不直接修改历史
- 作业调度如果上一次执行未完成，跳过本次执行
- 权限检查需要递归处理继承链
