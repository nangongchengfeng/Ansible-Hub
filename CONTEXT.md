# Ansible Job Platform

一个企业级 Ansible 自动化作业平台，提供主机管理、剧本编排、作业调度和审计分析功能。

## Language

### 核心概念

**作业模板（Job Template）**:
保存的可复用自动化作业定义，包含目标主机、执行动作和参数配置。

支持手动触发或周期调度（使用 Cron 表达式）。调度有 `enabled` 状态，可暂停/启用。如果上一次执行未完成，下一次触发时间到时跳过本次执行。
_Avoid_: 作业、模板

**作业执行（Job Execution）**:
作业模板或临时操作的一次具体运行实例，记录执行状态、输出日志和结果。

状态流转：`pending` → `running` → `success`/`failed`/`timeout`/`cancelled`。

可由创建者、运维或超级管理员取消正在执行的作业。
_Avoid_: 作业、任务、执行记录

**任务（Task）**:
Celery 或 Ansible 层面的单个执行单元，是作业执行的组成部分。
_Avoid_: 作业

**剧本（Playbook）**:
Ansible Playbook，定义一组自动化操作的编排文件，版本化管理。每次保存创建新版本，支持查看历史版本和diff对比，回滚时创建新版本。删除时级联删除所有历史版本。
_Avoid_: Playbook（英文术语在中文讨论中可以使用，但优先使用"剧本"）

**脚本（Script）**:
可复用的运维脚本（Shell、Python 等），版本化管理，可在作业中调用。每次保存创建新版本，支持查看历史版本和diff对比，回滚时创建新版本。删除时级联删除所有历史版本。
_Avoid_: 文件

### 资产与连接

**业务节点（Business Node）**:
树状业务关系中的一个节点，用于组织和分组主机。包含名称、描述、父节点引用和排序字段。允许为空（无主机），支持移动主机到其他节点。

支持按节点授权，细粒度权限包括 `view`（查看）、`execute`（执行作业）、`manage`（管理）。权限可继承，子节点自动继承父节点权限，但可覆盖。
_Avoid_: 业务关系、分组

**主机（Host）**:
被管理的服务器实例，包含资产信息和连接配置。

连接配置包括内网IP、外网IP、IP偏好、可覆盖的SSH端口、关联的系统用户，以及可选的网关。

状态包括：
- `enabled`：管理状态，是否允许在此主机上执行作业
- `connection_status`：最后一次连接的结果，不主动探测，仅在执行作业时更新

资产信息包括云厂商等。
_Avoid_: 服务器、实例

**系统用户（System User）**:
用于登录目标主机的 SSH 凭证，支持私钥认证和密码认证。私钥和密码均使用 Fernet 加密存储。暂不支持自定义 SSH 配置，使用合理默认值。
_Avoid_: 凭证、SSH 用户

**网关（Gateway）**:
SSH 跳板机/堡垒机，用于连接内网主机。
_Avoid_: 跳板机、堡垒机

### 安全与审计

**命令过滤规则（Command Filter Rule）**:
定义的高危命令拦截规则，支持字符串匹配和正则表达式。

- 字符串匹配：包含匹配，只要命令中包含该字符串即触发
- 正则表达式：匹配整个命令行
- 规则有 `enabled` 字段，可启用/禁用
- 规则有 `priority` 字段，高优先级规则先匹配，命中即阻止
_Avoid_: 过滤规则、规则

**审计日志（Audit Log）**:
记录平台所有写入操作（创建、更新、删除）以及作业执行的启动和完成的日志，不可删除，用于安全审计。仅审计员和超级管理员可查看。
_Avoid_: 日志、操作记录

## Relationships

- 一个 **业务节点** 可以包含多个 **主机** 和子 **业务节点**
- 一个 **主机** 属于 exactly one **业务节点**
- 一个 **主机** 使用 exactly one **系统用户** 进行连接
- 一个 **主机** 可以通过 zero or one **网关** 连接
- 一个 **作业模板** 可以产生多个 **作业执行**
- 一个 **作业执行** 包含多个 **任务**
- 一个 **剧本** 可以被多个 **作业模板** 使用
- 一个 **脚本** 可以被多个 **作业模板** 或 **剧本** 使用

## Example dialogue

> **Dev**: "当用户创建一个 **作业模板** 后，如何运行它？"
> **Domain expert**: "用户可以手动触发 **作业模板**，产生一个新的 **作业执行**，或者设置定时调度自动产生 **作业执行**。"

> **Dev**: "如果一个 **作业执行** 失败了，能重做吗？"
> **Domain expert**: "可以，可以基于失败的 **作业执行** 重新创建一个新的 **作业执行**，使用相同的参数。"

## Flagged ambiguities

- "作业"曾被用来表示 **作业模板** 和 **作业执行** 两个概念 — 已解决：拆分两个独立术语
