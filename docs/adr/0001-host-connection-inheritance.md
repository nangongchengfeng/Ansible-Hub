# 主机连接配置的继承策略

决定了主机连接配置（网关、SSH端口）的继承策略，以及IP地址的存储方式。

## 决策内容

- **IP地址**：主机同时存储 `内网_ip` 和 `外网_ip` 两个字段，通过 `ip_preference` 枚举（`internal`/`external`）选择使用哪个
- **SSH端口**：系统用户设置默认端口，主机可以单独覆盖
- **网关**：网关可以绑定到主机，也可以绑定到业务节点；主机优先使用自己绑定的网关，如果没有则继承所属业务节点的网关，逐级向上查找

## 考虑的替代方案

- **单IP字段**：只存一个IP，由用户决定填内网还是外网 — 被拒绝，因为灵活性不足，同一主机在不同场景可能需要用不同IP
- **网关只能绑定到主机**：不支持继承 — 被拒绝，因为大量主机使用同一网关时配置繁琐
- **网关只能绑定到业务节点**：不支持主机单独绑定 — 被拒绝，因为灵活性不足，同一业务节点下可能有例外

## 后果

- 配置更灵活，但连接时的解析逻辑稍复杂
- 支持大规模主机的批量配置（通过业务节点绑定网关）
- 也支持单个主机的例外配置
