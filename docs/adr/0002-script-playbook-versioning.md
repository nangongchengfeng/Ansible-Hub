# 剧本和脚本的版本控制策略

决定了剧本（Playbook）和脚本（Script）的版本控制、删除和回滚策略。

## 决策内容

- **版本存储**：使用独立的 `ScriptVersion` 和 `PlaybookVersion` 表，每次保存创建新版本，记录变更人、变更时间、变更说明，存储完整内容
- **删除策略**：删除剧本或脚本时，级联删除其所有历史版本
- **版本数量**：不限制保留的版本数量
- **回滚策略**：回滚到历史版本时，创建一个新版本（将历史内容复制为新版本），而不是直接修改当前版本

## 考虑的替代方案

- **删除时保留历史**：标记父对象已删除但保留历史 — 被拒绝，避免垃圾数据累积
- **限制版本数量**：只保留最近 N 个版本 — 被拒绝，审计可能需要查看完整历史
- **回滚直接替换**：回滚时直接把历史版本设为当前版本 — 被拒绝，因为回滚操作本身也应该有审计记录

## 后果

- 回滚操作可审计，有完整历史记录
- 删除操作简单，但历史数据不可恢复
- 存储使用量随时间增长，但对于文本内容来说通常可接受
