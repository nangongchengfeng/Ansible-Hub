# 业务节点的授权策略

决定了业务节点的权限模型和继承规则。

## 决策内容

- **权限类型**：细粒度权限，包括 `view`（查看）、`execute`（执行作业）、`manage`（管理主机/系统用户等）
- **继承规则**：权限可以继承，子业务节点自动继承父业务节点的权限
- **覆盖规则**：子业务节点可以设置自己的权限，覆盖从父节点继承的权限

## 考虑的替代方案

- **简单的可见/不可见**：不做细粒度权限划分 — 被拒绝，灵活性不足
- **不支持继承**：每个节点单独授权 — 被拒绝，大规模配置繁琐

## 后果

- 权限配置灵活，可以满足复杂的企业组织架构需求
- 权限计算逻辑稍复杂，需要递归查找继承链
